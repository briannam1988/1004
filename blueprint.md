
# Blueprint: AI 문구 생성기 V2.0

이 문서는 AI 문구 생성기 프로젝트의 기능, 디자인, 개발 계획을 정의하는 마스터 설계도입니다.

## 1. 프로젝트 개요

사용자의 다양한 상황과 요구에 맞춰 AI가 최적의 문구를 생성해주는 웹 애플리케이션입니다. 단순한 텍스트 생성을 넘어, 사용자의 생산성을 극대화하고 커뮤니케이션을 돕는 것을 목표로 합니다.

---

## 2. 개발 계획 로드맵

### **Phase 0: 초기 MVP 개발 (완료)**
- **목표:** 핵심 기능의 빠른 프로토타이핑 및 배포
- **주요 기능:**
  - 기본 UI/UX 구축 (상황, 톤, 추가 정보 입력)
  - Cloudflare AI 연동 및 서버리스 Functions 백엔드 구현
  - Cloudflare Pages를 통한 CI/CD 배포 자동화
  - 단일 결과물 생성 및 복사 기능
  - 로컬 스토리지를 이용한 기본 생성 기록 저장

### **Phase 1: 핵심 기능 강화 (진행중)**
- **목표:** 사용자에게 "Wow" 경험을 선사하는 압도적인 기능 구현
- **주요 기능:**
  - **UI/UX 개편:**
    - **'상황' 카테고리 세분화:** `<optgroup>`을 사용하여 보고, 요청 등의 하위 카테고리 제공
    - **'글의 대상' 선택 기능 추가:** `상사, 동료, 고객, 친구` 등 대상에 따른 뉘앙스 조절
    - **'글자 수' 조절 기능 추가:** `짧게(100자 내외), 중간(300자 내외), 길게(500자 이상)` 길이 선택
    - **결과물 카드 UI:** 3가지 버전의 결과물을 각각의 카드에 표시하여 비교 및 선택 용이성 증대
  - **백엔드 로직 강화:**
    - 새로운 입력(대상, 글자 수)을 모두 반영하여 AI 프롬프트를 고도화
    - AI가 3가지 결과물을 JSON 객체 내 배열(`{"generated_texts": [...]}`) 형태로 안정적으로 반환하도록 명령어 수정 및 파싱 로직 구현

### **Phase 2: 사용자 경험(UX) 업그레이드 (예정)**
- **목표:** 사용자가 서비스에 계속 머물고, 습관처럼 사용하게 만드는 편의 기능 추가
- **주요 기능:**
  - **고급 히스토리:** 입력값(상황, 대상, 톤 등)과 생성된 결과물 3종을 한 세트로 저장
  - **즐겨찾기:** 마음에 드는 결과물을 별도로 저장하고 언제든 다시 볼 수 있는 기능
  - **오늘의 글감 추천:** 글쓰기가 막막한 사용자를 위해 재미있는 주제를 제안하는 기능

### **Phase 3: 성장 및 확산 (예정)**
- **목표:** 더 많은 사용자를 유입시키고, 서비스가 자발적으로 퍼져나가게 하는 성장 동력 확보
- **주요 기능:**
  - **콘텐츠 마케팅용 블로그:** "보고서 잘 쓰는 법" 등 유용한 정보를 제공하여 검색 엔진 트래픽 유입
  - **결과물 공유 기능:** 생성된 문구를 클릭 한 번으로 `카카오톡`, `이메일` 등으로 공유하는 기능 추가
